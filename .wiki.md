# Project Summary
Acollya is a progressive web app (PWA) that leverages artificial intelligence to provide comprehensive mental health support. The platform allows users to track their moods, maintain an emotional diary, and interact with an AI therapist. It connects users to licensed mental health professionals and offers personalized self-care programs, along with analytics for visualizing emotional trends. Recent updates have focused on resolving issues with the production build, including the inclusion of the `dist` folder in downloads and the creation of a `_redirects` file for deployment on Netlify.

# Project Module Description
- **Onboarding**: Guides users through app features.
- **Authentication**: Manages user login, registration, and password recovery, including session management in Supabase and Google SSO integration.
- **Home/Dashboard**: Displays personalized greetings and quick actions.
- **Mood Check-in**: Allows users to log their feelings and receive AI-generated insights.
- **Emotional Diary**: Users document their thoughts and feelings.
- **Therapeutic Chat**: Engages users in conversation with AI, featuring message limits and indicators.
- **Self-care Programs**: Provides guided programs for emotional well-being.
- **Profile Management**: Users manage their accounts and preferences.
- **Settings**: Adjust app configurations.
- **Privacy and Security**: Details user data protection and rights.
- **Analytics**: Provides insights into emotional trends over time.
- **Therapist Integration**: Connects users with mental health professionals.
- **Appointment Management**: Users can schedule, manage, and view therapy sessions.
- **Payment Integration**: Manages subscriptions and payments through Stripe.
- **Analytics Tracking**: Monitors user interactions and engagement through Firebase.
- **Edge Functions**: Manages backend operations, including AI chat, mood insights, and journal reflections.

# File Description Inventory
- **components/**: Contains reusable UI components, including indicators for message limits and loading states.
- **contexts/**: Manages application state through context.
- **hooks/**: Custom hooks for state management.
- **lib/**: Utility functions and configurations for external services, including Edge Functions.
- **pages/**: Application pages, including onboarding and main features.
- **services/**: API service files for handling requests and responses, updated to utilize new Edge Functions.
- **types/**: TypeScript type definitions for various entities, including Edge Function requests and responses.
- **supabase/**: Contains Supabase schema and Edge Functions.
  - **functions/**: Functions for chat, mood insights, therapist matching, and journal management.
    - **mood-checkin/**: TypeScript implementation of the mood check-in functionality.
    - **chat-ai/**: TypeScript implementation for AI chat.
    - **journal-reflection/**: TypeScript implementation for journal reflections.
    - **therapist-matching/**: TypeScript implementation for therapist matching.
    - **mood-insights/**: TypeScript implementation for mood insights.
    - **program-progress/**: TypeScript implementation for tracking program progress.
    - **appointment-booking/**: TypeScript implementation for booking therapy appointments.
    - **create-checkout/**: TypeScript implementation for Stripe checkout sessions.
- **App.tsx**: Main entry point of the application.
- **README.md**: Documentation for project setup and usage.
- **.env.local**: Environment variables for the application.
- **EDGE_FUNCTIONS_DEPLOYMENT.md**: Guide for configuring Edge Functions.
- **SETUP_INSTRUCTIONS.md**: Documentation for final setup instructions and troubleshooting.

# Technology Stack
- **Frontend**: React 19, TypeScript, Vite, Tailwind CSS, Shadcn/ui
- **Backend**: Supabase (PostgreSQL, Auth, Edge Functions), OpenAI (AI insights), Stripe (Payment processing), Firebase (Analytics)

# Usage
### Installation
```bash
pnpm install
```

### Development
```bash
pnpm run dev
```

### Build for Production
```bash
pnpm run build
```

### Create `_redirects` File
Ensure the `_redirects` file is created in the `dist/` folder with the content: 
```
/*    /index.html   200
```

### Preview the Build
```bash
pnpm run preview
```

### Linting
```bash
pnpm run lint
```

### Configuration
- **Demo Mode**: Default mode using localStorage with no backend required.
- **Production Mode**: Requires configuration of Supabase, Stripe, Firebase, and OpenAI API keys.
