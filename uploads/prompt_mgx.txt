Quero que você crie uma aplicação web completa no formato de PWA (Progressive Web App), focada em mobile-first, chamada **Acollya**.

A Acollya é um aplicativo de suporte emocional com IA, voltado para o público brasileiro, que oferece:
- check-ins de humor,
- diário emocional (texto e áudio),
- programas de autocuidado baseados em Terapia Relacional Sistêmica e TCC,
- chatbot terapêutico com IA em português,
- e, em uma fase posterior, escalonamento para teleconsulta com psicólogos humanos.

IMPORTANTE SOBRE BACKEND / APIs
- Neste primeiro momento, quero que você crie APENAS o front-end PWA completo.
- TODO acesso a dados deve ser feito via chamadas a endpoints REST que eu vou implementar mais tarde em um backend próprio em FastAPI (Python).
- Isso significa:
  - Você deve declarar e usar funções de chamada de API (fetch/axios) para endpoints REST **futuros**, mas não precisa implementar servidor.
  - Onde fizer sentido, use mocks locais / dados fake para que a aplicação funcione visualmente, mas com a estrutura pronta para depois ligar nas APIs reais.
- Crie um arquivo central de configuração de API, por exemplo:
  - `src/config/api.ts` ou `src/services/apiClient.ts`
  - com algo como `BASE_URL = "https://api.acollya.com"` (pode ser placeholder).
- Todas as chamadas de rede devem usar esse cliente central.

## 1. Stack e requisitos técnicos do PWA

- Crie uma aplicação **PWA mobile-first**, que funcione muito bem em smartphones e também em desktop.
- Linguagem: TypeScript, se possível.
- Framework sugerido: React (com Next.js ou outra solução que você julgar adequada para PWA) ou outro stack moderno equivalente. O importante é:
  - suporte a PWA (service worker, manifest),
  - código organizado e componentizado.
- Estrutura:
  - `src/components` para componentes reutilizáveis,
  - `src/pages` ou equivalente para as telas,
  - `src/hooks` para hooks customizados,
  - `src/services` para chamadas de API,
  - `src/state` ou `src/store` para gerenciamento de estado global (pode usar Zustand, Redux ou Context API, o que você preferir).
- Implemente:
  - Manifest PWA (nome, ícone, theme color),
  - Service worker para cache de assets básicos,
  - Comportamento de “Add to Home Screen”.
- Considere **offline-first básico**:
  - Se o usuário estiver offline, exiba estados amigáveis (ex: “Você está offline. Vamos salvar este conteúdo localmente e sincronizar depois.”),
  - Permita armazenar temporariamente check-ins e diários localmente (localStorage ou IndexedDB) e marcar para sincronização futura.

## 2. Identidade Visual e UI/UX (muito importante)

Use o seguinte kit visual:

### Paleta primária
- **Lavanda Serenidade**: `#C9C1F0` (fundos calmos, áreas de alívio visual)
- **Azul Sálvia Inteligente**: `#6A80A6` (textos principais, ícones, títulos)
- **Off-White Acolhedor**: `#F5F4F8` (background geral)
- **Lavanda Profunda**: `#7E71C9` (botões principais / CTAs)

### Paleta secundária
- **Pêssego Humano**: `#F3D5C4` (toques de afeto, cartões de feedback positivo)
- **Verde Névoa**: `#B9D6C2` (estado de equilíbrio, sucesso suave)

### Tipografia
- Headings: Sofia Sans / Nunito / Avenir Rounded (escolha a que tiver disponível, com vibe arredondada e acolhedora).
- Corpo de texto: Inter / SF Pro / Lato.
- Tamanhos mobile:
  - H1: 32px semibold
  - H2: 26px semibold
  - H3: 22px medium
  - Body1: 17px
  - Body2: 15px
  - Caption: 13px

### Componentes (estilo)

- **Botão primário**:
  - Fundo `#7E71C9` (Lavanda Profunda),
  - Altura ~54px,
  - Raio 12px,
  - Sombra suave,
  - Micro-animação de pulsação sutil ao passar o mouse / tap (scale pequeno, suave).
- **Botão secundário**:
  - Sem fundo,
  - Borda em Lavanda Profunda,
  - Texto em Azul Sálvia.
- **Botão terciário**:
  - Apenas texto em Azul Sálvia, sem borda, estilo link.

- **Chat bubbles**:
  - Bolha da IA:
    - Fundo Lavanda Serenidade `#C9C1F0`,
    - Texto em Azul Sálvia `#6A80A6`.
  - Bolha do usuário:
    - Fundo Lavanda Profunda `#7E71C9`,
    - Texto branco.
  - Raio de borda: ~18px,
  - Padding interno: ~16px.

- **Inputs**:
  - Fundo branco,
  - Borda cinza neutro `#CECED8`,
  - Raio 12px,
  - Altura 48px,
  - Placeholder em Cinza Calmo `#A3A3AF`.

- **Cards**:
  - Fundo Off-White `#F5F4F8`,
  - Sem borda,
  - Sombra muito suave,
  - Raio 22px.

- **Ícones**:
  - Estilo line-art de 2px,
  - Formas orgânicas e arredondadas,
  - Ícones: chat, coração, diário, emoções, escudo de privacidade, relógio, etc.
  - Cores: Azul Sálvia ou Lavanda Profunda.

### Layout / Grid / Microinterações

- Grid mobile: 4 colunas, gutter de 16px.
- Padding horizontal das telas: 24px.
- Espaçamento entre seções: 40–56px.
- Use bastante espaço em branco para sensação de calma.
- Microinterações:
  - Animação de “breathe” (crescer/diminuir suave) no ícone principal ou no logo em tela de boas-vindas.
  - Transições entre telas com fade + slide suave.
  - Chat com indicador de digitação (3 pontos pulsando).
  - CTAs com pulsação leve quando entram na viewport.

### Tom de texto na interface
- Sempre em **português brasileiro**.
- Linguagem:
  - Acolhedora, humana, empática, direta,
  - Sem jargões clínicos pesados,
  - Mistura de “amigo confiável” e “mentor sereno”.

## 3. Estrutura de Navegação e Páginas

Crie um fluxo de navegação pensando em app mobile:

### 3.1. Onboarding + Boas-vindas
- Tela 1: Introdução à Acollya, explicando que é suporte emocional com IA + profissionais humanos.
- Tela 2: Benefícios (check-ins, diário, chat 24/7, programas).
- Tela 3: Mensagem sobre privacidade, acolhimento sem julgamentos, LGPD.
- Botão “Começar” leva para tela de login/cadastro.

### 3.2. Autenticação
- Tela de login:
  - E-mail + senha,
  - Botão “Entrar”,
  - Botão “Continuar com Google”,
  - Link “Criar conta”.
- Tela de cadastro:
  - Nome, e-mail, senha,
  - Checkbox para aceitar Termos de Uso e Política de Privacidade,
  - Botão “Criar conta”.
- Tela de esqueci a senha:
  - Campo e-mail,
  - Orientações simples.

**Integração com futuras APIs**:
- Login: `POST /auth/login`
- Cadastro: `POST /auth/register`
- Login social (Google): `POST /auth/google`
Use funções de `apiClient` chamando esses endpoints (placeholder).

### 3.3. Home / Dashboard Emocional
- Saudações personalizadas: “Oi, [nome]! Como você está hoje?”
- Seções principais em cards:
  - “Fazer check-in de humor”
  - “Escrever no diário”
  - “Falar com a Acollya (chat IA)”
  - “Ver programas de autocuidado”
- Mini painel com:
  - Gráfico simples de humor dos últimos 7 dias,
  - Recomendações baseadas no histórico (mock no início).

Dados via futuras APIs:
- `GET /me`
- `GET /mood-checkins/summary`
- `GET /recommendations`

### 3.4. Check-in de Humor
- Tela com fundo em degradê suave lavanda/off-white.
- Ícones de emoções (ex: Muito bem, Bem, Neutro, Triste, Muito triste).
- Campos:
  - Emoção principal (obrigatório),
  - Emoções secundárias (opcional, múltipla escolha),
  - Texto livre (“Quer contar um pouco mais?”).
- Botões:
  - Primário: “Registrar meu momento”
  - Secundário: “Pular por enquanto”
- Após registrar, mostrar:
  - Mensagem acolhedora,
  - Sugestão de ir para diário ou chat se a emoção estiver negativa.

APIs futuras:
- `POST /mood-checkins`
- `GET /mood-checkins` (para histórico)

### 3.5. Diário Emocional
- Lista de entradas de diário (data, trecho do texto).
- Botão “Nova entrada”.
- Tela de nova entrada:
  - Campo texto grande,
  - Botão para gravar áudio (no front, simular apenas; backend futuro poderá receber arquivo).
  - Botão “Salvar diário”.
- Após salvar, chamar IA para sugerir reflexão (futuro: via backend).
  - Chamada front → `POST /ai/journal-reflection` (endpoint futuro).

APIs futuras:
- `POST /journals`
- `GET /journals`
- `POST /ai/journal-reflection`

### 3.6. Chat IA Terapêutico
- Tela de chat estilo mensageiro.
- Lista de mensagens (user/IA, com bolhas conforme design).
- Campo de texto + botão enviar.
- Possibilidade futura de enviar áudio/imagem (crie o UI já, mesmo que inicialmente não funcione 100%).
- Indicador de IA digitando.
- Mensagens iniciais:
  - IA se apresenta, explica que é IA de suporte e não substitui terapia.
- Regras de UX:
  - Mostrar mensagens claras, empáticas e curtas, com opção “ver mais” se alongar.
  - Incluir avisos na interface em algum lugar fixo: “A Acollya não substitui terapia presencial nem atendimento de emergência. Em caso de risco imediato, ligue 188 (CVV).”

APIs futuras:
- `POST /chat/send`
  - Body: `{ sessionId, message, userContext }`
- `GET /chat/sessions/:id`
- `GET /chat/sessions` (lista de sessões anteriores)

Para MVP, você pode usar mock de respostas da IA (ex.: mensagens estáticas) mas já deixe a chamada preparada para essas rotas.

### 3.7. Programas de Autocuidado
- Tela listando programas (ex: “21 dias de comunicação consciente”, “7 dias para respirar melhor”).
- Cada programa em card:
  - Nome, tempo (7/14/21 dias), breve descrição, status (não iniciado / em andamento / concluído).
- Ao clicar:
  - Tela com etapas diárias (Dia 1, Dia 2, ...),
  - Cada dia com:
    - texto breve, áudio ou vídeo (mock no início),
    - exercícios (perguntas/reflexões).
- Marcar dia como concluído.

APIs futuras:
- `GET /programs`
- `GET /programs/:id`
- `POST /programs/:id/progress`

### 3.8. Painel de Emoções (Analytics do Usuário)
- Tela com gráfico de linha (humor ao longo do tempo).
- Segmentação:
  - Últimos 7 dias,
  - Últimos 30 dias.
- Cards com insights (mock):
  - “Você tem se sentido mais ansiosa à noite.”
  - “Dia da semana mais leve: sábado.”

APIs futuras:
- `GET /analytics/mood`
- `GET /analytics/recommendations`

### 3.9. Integração com Profissionais (Fase futura)
Mesmo que as APIs ainda não existam, crie a estrutura de UI:

- Tela “Encontre um psicólogo”:
  - Lista de profissionais com foto/ícone, nome, abordagem, faixa de preço.
- Tela de detalhes:
  - Bio curta, CRP, áreas de atuação, botões “Ver horários”.
- Agenda:
  - Calendário simples com slots de horário clicáveis.
- Ao escolher horário, abre fluxo de agendamento (mock) com confirmação e aviso que o recurso estará disponível em breve.

APIs futuras:
- `GET /therapists`
- `GET /therapists/:id`
- `GET /therapists/:id/availability`
- `POST /appointments`

### 3.10. Perfil, Configurações e Conta
- Tela “Perfil”:
  - Nome, e-mail,
  - Preferências de notificação,
  - Botão “Gerenciar plano”.
- Tela “Privacidade e segurança”:
  - Botão para solicitar exclusão de conta,
  - Link para Termos de Uso e Política de Privacidade.
- Tela “Sobre a Acollya”:
  - Missão, visão, valores.

APIs futuras:
- `GET /me`
- `PUT /me`
- `DELETE /me` (exclusão de conta)

### 3.11. Assinaturas (pode ser placeholder visual)
- Tela que explica:
  - Plano gratuito vs plano premium,
  - Benefícios de cada um.
- Botões “Assinar plano premium”.
- Por enquanto, deixe apenas como tela de apresentação com botão que exibe um modal dizendo que será ativado em breve.

APIs futuras:
- `GET /subscriptions/current`
- `POST /subscriptions/upgrade`

## 4. LGPD, Ética e Mensagens Importantes (apenas front-end)

- Inclua desde o MVP:
  - Tela ou modal com Termos de Uso e Política de Privacidade.
  - Avisos no chat:
    - “A Acollya é um apoio emocional e não substitui terapia com profissional nem serviço de emergência.”
  - Se o usuário escrever palavras ligadas a autoagressão (você pode apenas simular um caso com regex no front):
    - Exibir uma mensagem especial na UI (ex: modal) orientando:
      - a procurar ajuda profissional,
      - a ligar 188 (CVV) em caso de risco imediato.
    - Essa lógica será refinada no backend mais tarde, mas você já pode estruturar o front.
